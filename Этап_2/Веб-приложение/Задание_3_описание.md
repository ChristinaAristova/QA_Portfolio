# Задание 3: Тестирование личного кабинета кандидата и компании

## Описание проекта

Углубленное функциональное и кросс-браузерное тестирование личных кабинетов двух типов пользователей (кандидата и компании) на платформе **Hitch**.

### Цель

Применить техники тест-дизайна (граничные значения, классы эквивалентности) для проверки валидации полей, работы с данными и загрузки файлов, а также обеспечить корректную работу интерфейса в разных браузерах.

### Используемые инструменты

*   **YouGile** (система баг-трекинга)
*   **Google Документы/Таблицы** (тест-кейсы, чек-листы)

### Ключевые результаты и достижения

*   **Закрепление техник тест-дизайна:** Успешное применение граничных значений и классов эквивалентности на практике при тестировании сложных форм личных кабинетов.
*   **Кросс-браузерное тестирование:** Проведено тестирование в двух популярных браузерах для обеспечения стабильности и совместимости интерфейса (использовался сервис статистики браузеров для выбора целевых платформ).
*   **Комплексная проверка:** Составлены тест-кейсы, охватывающие позитивные/негативные сценарии, проверку предзаполненных данных, валидацию текстовых/числовых полей и загрузку файлов.
*   **Обратная связь:** Работа принята преподавателем.

### Освоенные навыки

*   Проведение кросс-браузерного тестирования.
*   Углубленное применение техник тест-дизайна на сложном функционале.
*   Оценка валидации различных типов полей (числовые, текстовые, ссылки, файлы).
*   Планирование и ведение тестовой документации для двух связанных модулей.
