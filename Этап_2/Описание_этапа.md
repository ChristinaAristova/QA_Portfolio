# Этап 2: Функциональное тестирование UI/UX

Проект выполнен на сайте по поиску аутстаф-специалистов **Hitch**. Количество заданий: 5 из 9.

**Ключевые навыки и достижения:**

*   **Исследовательское и интуитивное тестирование:** Применены для быстрой оценки продукта, сбора тест-кейсов и поиска дефектов "на ходу".
*   **Системы баг-трекинга:** Освоена работа с **YouGile** для создания баг-репортов, управления ими и контроля дефектов.
*   **Тестирование UI и форм:** Расширены знания по тестированию пользовательского интерфейса, включая различные UI-элементы и формы.
*   **Техники тест-дизайна:** Применены для структурирования процесса тестирования и повышения его эффективности.
    *   **Классы эквивалентности:** Использованы для деления данных на группы и сокращения количества тестов.
    *   **Граничные значения:** Применены для выявления ошибок, которые чаще всего возникают на границах диапазонов.
    *   ~~**Попарное тестирование:**~~ *[В процессе изучения]* Сокращение числа тестов за счет комбинации параметров парами.

*   ~~**Предугадывание ошибок (Error Guessing):**~~ *[В процессе изучения]* Использование пользовательского опыта для целенаправленного поиска слабых мест в продукте.
*   **Тестирование требований:** Проверка требований на четкость и подробность для выявления потенциальных проблем на ранних этапах.
*   **Тестовая документация:** Изучены ключевые аспекты, включая определение метрик качества, составление тест-плана и написание отчета о тестировании.
*   ~~**Тестирование мобильных приложений:**~~ *[В процессе изучения]* Освоение специфики тестирования под мобильные платформы (эмуляторы, адаптивность).
